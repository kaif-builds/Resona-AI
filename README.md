# Resona AI

Resona AI is an AI-powered voice cloning platform that lets creators generate high-quality, realistic voice clones using a React/TypeScript frontend and a Python FastAPI backend powered by the Coqui XTTS-v2 model.

---

## Repository File Map

Below is a complete listing of every file in the repository, organised by directory, with a description of what each file does.

---

### Root-level files

| File | Purpose |
|------|---------|

| `metadata.json` | AI Studio app registry manifest. Declares the app name, description, and the `microphone` frame permission required by the recording feature. |
| `package.json` | NPM project configuration. Defines the `dev`, `build`, `preview`, `clean`, and `lint` scripts; lists all dependencies including React 19, Vite 6, Tailwind CSS 4, `@google/genai`, `lucide-react`, `motion`, and server-side utilities (`express`, `better-sqlite3`, `dotenv`) that support optional local API/server functionality. |
| `package-lock.json` | Locked, exact dependency versions generated by `npm install`. Ensures reproducible installs across environments. |
| `tsconfig.json` | TypeScript compiler configuration. Targets ES2022, uses ESNext modules, enables `react-jsx` transform, experimental decorators, and maps the `@/*` import alias to the repository root. |
| `vite.config.ts` | Vite bundler configuration. Registers the React and Tailwind CSS plugins, injects `GEMINI_API_KEY` from the environment, configures the `@` path alias, and optionally disables HMR via `DISABLE_HMR`. |
| `.env.example` | Template for required environment variables (`GEMINI_API_KEY`, `APP_URL`). Copy to `.env` and fill in real values before running locally. |
| `.gitignore` | Git ignore rules. Excludes `node_modules/`, `dist/`, `build/`, `coverage/`, log files, and `.env*` files (while keeping `.env.example`). |
| `README.md` | This file. Documents the project, its architecture, and the function of every file in the repository. |

### `/src` — React / TypeScript frontend source

| File | Purpose |
|------|---------|
| `src/main.tsx` | React entry point. Wraps `<App>` in `StrictMode` and mounts it to the `#root` DOM node using React 19's `createRoot` API. |
| `src/App.tsx` | Main application (738 lines). See the **App.tsx components** subsection below for a breakdown of each component. |
| `src/index.css` | Global stylesheet. Imports Tailwind base and component layers; declares Inter and JetBrains Mono web fonts; defines reusable utility classes: `.cinematic-bg` (fixed radial-gradient background with blurred image overlay), `.grain-overlay` (subtle noise texture at 6 % opacity), `.glass-panel` (backdrop-blur glass-morphism card), `.nav-link` (navigation link with animated arrow prefix), `.search-input` (search field with emerald focus ring), and `.custom-cursor-container`. |
| `src/resonaApi.js` | Frontend API client library. Exports five functions that communicate with the FastAPI backend: `uploadVoiceSample`, `synthesizeSpeech`, `listProfiles`, `deleteProfile`, and `getProfileSampleUrl`. Includes error parsing with fallbacks for different HTTP error formats. |

#### App.tsx components

| Component | Function |
|-----------|----------|
| `App` | Root component. Manages global state (active view, mouse position, cursor trail, mobile breakpoint). Renders either the animated landing page or `VoiceCloningPage`. Implements a 3D tilt parallax effect with Framer Motion spring values and a music-note cursor trail. |
| `VoiceCloningPage` | Full voice recording and synthesis UI. Left panel: browser-based audio recording via `getUserMedia` with a live timer. Right panel: file upload supporting `.wav`, `.mp3`, `.ogg`, `.flac`, `.m4a`. Includes a status bar and a "Generate" button that activates once audio is ready. |
| `NavLink` | Styled navigation anchor with hover colour and underline animation. |
| `FooterSocialIcon` | Animated social-media icon; lifts upward and turns emerald green on hover. |
| `SideNavLink` | Navigation link with an icon and label; slides right on hover. |
| `ParticleBackground` | Canvas-based ambient animation. Renders 50 randomly positioned particles that move continuously and wrap around screen edges (drawn at z-index −10, 40 % opacity). |


### `/backend` — Python FastAPI voice-cloning engine

| File | Purpose |
|------|---------|
| `backend/main.py` | FastAPI application (279 lines). Lazily loads the Coqui XTTS-v2 model and exposes the REST API described in `backend/README.md`. Key helpers: `_convert_to_wav()` (FFmpeg WAV conversion), `_get_audio_duration()` (WAV duration with ffprobe fallback), `_cleanup_file()` (async delayed file deletion). |
| `backend/requirements.txt` | Python dependency list. Pins FastAPI 0.111.0, Uvicorn 0.29.0, Coqui TTS 0.22.0, PyTorch 2.2.2 (CPU-only), TorchAudio 2.2.2, librosa 0.10.2, soundfile 0.12.1, numpy 1.26.4, pydantic 2.7.1, python-multipart 0.0.9, and httpx 0.27.0. |
| `backend/railway.toml` | Railway deployment configuration. Specifies the nixpacks builder with `ffmpeg` and `python311` system packages, the start command (`uvicorn main:app --host 0.0.0.0 --port $PORT`), a `/health` healthcheck with a 300 s timeout, on-failure restart policy, and `PYTHONUNBUFFERED=1`. |
| `backend/resonaApi.js` | Extended frontend API client intended to be copied into `src/`. Provides the same helpers as `src/resonaApi.js` and adds two utilities: `downloadAudio(blobUrl, filename)` — triggers a browser download; `synthesizeSpeech(text, profileId, language)` — fetches audio and returns an object URL. **Note**: this file is the canonical, fuller version; `src/resonaApi.js` is a trimmed copy for the deployed frontend. Keep them in sync when modifying the API. |
| `backend/setup.bat` | Windows one-time environment setup script. Verifies Python 3.11 is available, creates a `venv` virtual environment, activates it, upgrades pip, and installs all packages from `requirements.txt`. |
| `backend/start-backend.bat` | Windows development launch script. Activates the `venv` created by `setup.bat` and starts the backend with `uvicorn main:app --reload --port 8000` (hot-reload enabled). |
| `backend/README.md` | Backend-specific documentation (164 lines). Explains how XTTS-v2 zero-shot cloning works, provides step-by-step local setup instructions for Windows (`.bat` files) and Linux/macOS (manual commands), documents the full REST API, describes how to connect the frontend via `resonaApi.js`, and notes operational constraints (first-boot latency ~2–3 min, CPU-only inference, ephemeral storage on Railway free tier). |

## Quick Start

```bash
# Frontend
npm install
npm run dev          # http://localhost:3000

# Backend (Windows)
cd backend
setup.bat
start-backend.bat    # http://localhost:8000

# Backend (Linux/macOS)
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```
